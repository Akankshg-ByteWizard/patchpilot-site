GitHub Pages Checklist

Use this checklist before every release push to keep deployment stable and links healthy.

  1. Repository root: keep HTML files at project root with asset paths like assets/styles.css.
  2. Branch: push to main branch and ensure static files are committed.
  3. Pages settings: in GitHub, set Pages source to Deploy from branch and choose main and root.
  4. Verify links: confirm navigation works for every page after deployment.
  5. Custom domain: optional, add CNAME once base site is stable.
